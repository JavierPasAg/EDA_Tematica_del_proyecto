# EDA-Oferta_Turistica_vs_Precios_Madrid

## Introducción
El objetivo de este proyecto es la asimilación y puesta en práctica de los conocimientos obtenidos en el Bootcamp de Data Science Online de The Bridge.
Se trata de un proyecto colaborativo en el que vamos a elaborar un EDA, planteando un problema al que queremos dar una justificación o respuesta apoyándonos en datos que lo respaldan.

## Descripción breve

Nuestro EDA busca dar respuesta a la pregunta actual de si el aumento de la oferta de pisos de alquiler turístico afecta o guarda relación directa con el aumento de los precios de acceso a la vivienda, tanto en modalidad de alquiler como compra. Por ello, apoyados en datos de portales referencia en el sector, queremos poner de manifiesto si existe esta relación directa entre el auge de la oferta turística y el aumento de los precios.
Hemos tomado como ciudad de referencia Madrid ya que concentra mucha oferta turística y nos ofrece gran cantidad de datos para poder sacar conclusiones que justifiquen esta problemática.

## Hipótesis planteada
La principal hipótesis que queremos respaldar con los datos analizados es que el aumento de la densidad turística afecta directamente elevando los precios de la vivienda debido a varias razones que queremos demostrar:
   - Creemos que en los barrios con mayor densidad de oferta turística, presentan un mayor precio por metro cuadrado, lo que eleva los precios de acceso a la vivienda tanto en alquiler como en compra.
   - Esta demanda elevada de alquiler para uso turístico o estacional, permite que los propietarios puedan establecer precios más elevados y obtener un beneficio mayor que con el alquiler tradicional como vivienda habitual, lo que hace que esta oferta se reduzca y encarezca los disponibles en esta modalidad.

## Tecnologías utilizadas
Para la elaboración de este EDA, hemos trabajado con el IDE Visual Studio Code haciendo uso del lenguaje python y notebooks para la presentación y tratamiento de los datos.
Como librerías hemos empleado pandas, numpy, requests, seaborn y matplotlib, entre las principales.

Para la extracción de datos, hemos intentado hacer uso de la API de los distintos portales inmobiliarios y su propio sitio web.
Para la elaboración de la documentación, memoria y presentación, haremos uso de archivos con extensión .pdf y pptx para una mejor legibilidad y estructuración.
Por último, para la presentación y las reuniones que hemos mantenido, hemos utilizado google meet para permitirnos la comunicación directa.

## Estructura del repositorio
El respositorio tiene la siguiente estructura:
- Un nivel raíz en el que tenemos una carpeta de recursos (src), la Memoria, la presentación, este fichero Readme y el notebook principal.
- Dentro de la carpeta src almacenamos todos los recursos que hemos necesitado para la elaboración del EDA, como los ficheros de datos que obtuvimos de las distintas fuentes.

## Instrucciones de reproducción
La forma de reproducir el código es sencilla ya que englobamos todo lo ejecutable en un mismo notebook para que pueda ser reproducido y ejecutado de forma secuencial, mostrando los resultados esperados y comentando los pasos y análisis con celdas de markdown.
